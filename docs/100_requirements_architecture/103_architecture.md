## 3. 数据流

```text
用户操作
  ↓
页面层（捕获事件，展示结果）
  ↓
业务层（校验、编排、调用）
  ↓
数据层 / 基础设施层（读写、日志、网络）
  ↓
返回结果
```

## 4. 边界规则

| 规则 | 说明 |
|------|------|
| 页面层不直接操作数据库 | 所有数据操作必须经过业务层 → 数据层 |
| 业务层不直接写 console.log | 必须通过基础设施层的统一 logger |
| 公共能力放在基础设施层 | 被两个以上模块使用的能力必须下沉 |
