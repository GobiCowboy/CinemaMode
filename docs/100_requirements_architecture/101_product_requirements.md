# 101 产品需求

## 1. 产品名称

Cinema Mode for Mac

## 2. 产品定位

Cinema Mode 是一个轻量级 macOS 工具，让普通 Mac 用户在观看视频、课程、本地影片或直播时，通过一次点击进入低干扰观影状态。

应用在菜单栏保留一个极简图标作为入口，进入后提供一个始终可见的退出浮窗。

核心理念：让用户忘记电脑存在。

## 3. 目标用户

| 用户类型 | 使用场景 | 主要痛点 |
|----------|----------|----------|
| 普通 Mac 用户 | YouTube、Netflix、B 站、网课、直播等网页视频 | 全屏、菜单栏、Dock、退出路径之间切换麻烦 |
| 本地视频用户 | 使用 IINA、VLC、QuickTime 等播放器观看本地视频 | 想快速进入沉浸状态，不想记快捷键 |
| 轻度效率用户 | 临时需要把当前屏幕变成低干扰显示状态 | 不想理解系统设置、权限或自动化规则 |

## 4. 要解决的问题

- 手动进入沉浸状态步骤多，用户经常需要操作播放器、全屏按钮、系统设置或快捷键。
- 菜单栏和 Dock 在观看时持续占据视觉注意力。
- 退出沉浸状态不自然，用户可能忘记快捷键或找不到入口。
- 复杂自动识别、浏览器插件、播放器适配会增加认知负担，不适合 MVP 验证。

## 5. 设计原则

| 原则 | 说明 |
|------|------|
| 少一步 | 用户只需要点击一次，不需要学习、配置、初始化或记忆复杂快捷键。 |
| 用户主动决定 | 不识别视频、不监听播放器、不猜测场景；用户自己决定何时开始观影。 |
| 退出永远可见 | 隐藏系统 UI 后必须保留一个始终存在的退出入口。 |
| 不教育用户 | 不向普通用户解释系统原理、权限机制或技术细节。 |
| 产品适应用户 | 复杂性由产品承担，不要求用户调整习惯。 |

## 6. 本期要做

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| 301 | 进入观影模式 | 用户点击菜单栏图标后隐藏菜单栏和 Dock，并创建退出浮窗。 | P0 |
| 302 | 浮窗退出入口 | 默认右下角显示圆形极简浮窗，支持拖动，随鼠标状态调整透明度。 | P0 |
| 303 | 退出并恢复 | 用户点击浮窗后恢复进入前的系统显示状态并关闭观影模式。 | P0 |

## 7. 本期不做

| 功能 | 不做原因 | 后续 |
|------|----------|------|
| 自动识别视频播放 | 需要监听应用或网页状态，复杂度高，且偏离用户主动原则。 | 待评估 |
| AI 场景识别 | 对 MVP 价值验证帮助有限，会增加成本和解释负担。 | 待评估 |
| 浏览器插件联动 | 需要跨产品安装和权限说明，不符合一键原则。 | 待评估 |
| 播放器适配 | MVP 不绑定播放器，先验证通用沉浸入口是否成立。 | 待评估 |
| 网站识别 | 会引入隐私和维护成本。 | 待评估 |
| 专注模式联动 | 不是核心观影体验，且可能改变用户系统状态。 | 待评估 |
| 自动进入观影模式 | 容易误判，不符合用户主动决定原则。 | 待评估 |
| 复杂快捷键配置 | 增加学习成本。 | 永不作为默认体验 |

## 8. 成功标准

| 标准 | 怎么验证 |
|------|----------|
| 第一次打开即可理解 | 用户看到入口后无需阅读说明即可进入观影模式。 |
| 一次点击进入 | 从普通状态进入观影模式只需要一次明确点击。 |
| 退出入口始终存在 | 观影模式下任意时刻都有可点击退出浮窗。 |
| 菜单栏入口清晰 | 未进入观影模式时，菜单栏上有明确入口。 |
| 鼠标静止时低干扰 | 鼠标静止后浮窗透明度约 5%，不明显干扰内容。 |
| 鼠标移动时可发现 | 鼠标移动后浮窗透明度约 70%，用户知道入口存在。 |
| 鼠标悬停时可点击 | 悬停浮窗后透明度 100%，点击区域清晰。 |
| 退出后恢复原状态 | 菜单栏、Dock 和应用状态恢复到进入前。 |
| 用户愿意持续使用 | MVP 发布后观察下载、留存、推荐和日常使用反馈。 |

## 9. 需求验证问题

| 用户反馈 | 判断 |
|----------|------|
| “我直接按全屏就行了。” | 需求可能不成立。 |
| “我每天都在用。” | 需求成立，进入下一阶段迭代。 |
| “我不知道怎么退出。” | 设计失败，必须优先修复。 |
| “我不知道它在干什么。” | 入口表达失败，必须简化。 |
