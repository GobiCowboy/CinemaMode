# 202 项目目录结构

## 1. 实际目录结构

```text
project-root/
  src/                   # 源代码
    pages/               # 页面层
    services/            # 业务层
    data/                # 数据层
    infrastructure/      # 基础设施（logger、config、network…）
  docs/                  # 项目文档（按 docs/ 规范组织）
  tests/                 # 测试代码
```

> 不强制要求与上述完全一致，但必须有分层，且基础设施和业务代码物理隔离。

## 4. 禁止事项

- 不新增无说明的目录
- 不把业务逻辑写进页面代码
- 不绕过 logger 直接用 console.log
- 不把临时文件提交到 src/tests/docs 目录
