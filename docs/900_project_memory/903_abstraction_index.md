## 记录规则

- 第一次出现：可以不记录，除非明显会复用。
- 第二次出现：必须记录到本文件。
- 第三次出现：必须评估是否抽象。
- MVP 已明确会被 2 个以上功能复用的能力，开发前直接规划为公共模块。

## 抽象处理方式

1. 抽象成公共函数。
2. 抽象成公共组件。
3. 抽象成公共服务。
4. 暂不抽象时，必须在表格「建议处理」列写明原因。
